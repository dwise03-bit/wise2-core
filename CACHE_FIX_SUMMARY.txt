================================================================================
THE PROBLEM
================================================================================

Browsers cached old JavaScript chunks from previous build. When new website
was deployed with new HTML (containing glassmorphism/OUR WORKS section),
browsers continued to use old cached JavaScript.

Result: Hydration mismatch - new HTML couldn't work with old JavaScript.
Website showed old design despite successful deployment.

================================================================================
THE SOLUTION
================================================================================

Implemented nginx cache-busting headers at 3 levels:

1. JavaScript chunks (_ next/static/chunks/*):
   Cache-Control: public, max-age=31536000, immutable
   → Safe to cache 1 year because filename includes content hash

2. Static assets (_next/static/*):
   Cache-Control: public, max-age=31536000
   → Versioned by Next.js build process

3. HTML pages (/):
   Cache-Control: public, max-age=0, must-revalidate
   → Ensures browser always fetches fresh HTML on every visit

Result: New deployments always match HTML to correct JavaScript chunks.
