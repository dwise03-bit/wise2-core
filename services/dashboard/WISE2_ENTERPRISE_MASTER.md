## DESIGN LANGUAGE

### WISE² "Organized Chaos" Identity

**Color Palette**:
- **Primary Black**: #000000 (backgrounds, text)
- **Chrome Accent**: #E8E8E8 (borders, highlights)
- **Electric Blue**: #00D9FF (primary actions, links, focus)
- **Purple Accent**: #B300FF (secondary, emphasis)
- **Red Alert**: #FF0040 (errors, warnings, critical)

**Aesthetic**:
- Industrial, Cyberpunk, Premium Enterprise
- HUD overlays and blueprint graphics
- Glass panels with transparency
- Animated waveforms and visualizations
- Studio lighting aesthetic
- Dark mode primary (light mode secondary)

**Motion**:
- Motion-first design philosophy
- Smooth transitions (200-300ms)
- Reactive micro-interactions
- Loading state animations
- Visualization feedback

**Principles**:
- Responsive across all devices
- Accessible (WCAG 2.1 AA minimum)
- Performance-conscious (Core Web Vitals optimized)
- High contrast for legibility
- Consistent spacing & typography

## TECHNOLOGY STACK (LOCKED)

### Frontend
- **Framework**: Next.js 14+
- **Language**: TypeScript
- **UI Library**: React 19
- **Styling**: Tailwind CSS + CSS Modules
- **Components**: shadcn/ui base components
- **Animation**: Framer Motion
- **Audio**: Web Audio API, Tone.js
- **State**: React Context / TanStack Query
- **Forms**: React Hook Form
- **Real-time**: Socket.IO client

### Backend
- **Runtime**: Node.js 20+
- **Framework**: NestJS OR Express.js
- **Language**: TypeScript
- **API Style**: REST (JSON)
- **Real-time**: Socket.IO server
- **Authentication**: JWT + OAuth2
- **Validation**: Zod/Joi
- **Logging**: Pino

### Data
- **Relational DB**: PostgreSQL 15+
- **Cache**: Redis 7+
- **Search**: PostgreSQL Full-Text (or Elasticsearch)
- **Object Storage**: AWS S3 OR Google Cloud Storage OR MinIO
- **Message Queue**: Redis Streams OR RabbitMQ (async jobs)

### AI & ML
- **Primary**: Claude API (Anthropic)
- **Fallback**: Local Ollama (optional)
- **Orchestration**: Custom Node.js service
- **Agents**: Hermes-based multi-agent system
- **Tools**: LangChain or native implementations
- **Embeddings**: OpenAI or Anthropic API

### Infrastructure
- **Container**: Docker & Docker Compose
- **Orchestration**: Kubernetes (optional, for scale) OR Docker Swarm
- **Reverse Proxy**: Traefik OR Nginx
- **Monitoring**: Prometheus + Grafana
- **Logging**: ELK Stack OR Loki
- **CI/CD**: GitHub Actions
- **Hosting**: AWS / DigitalOcean / Hetzner
- **CDN**: CloudFlare OR Cloudfront

### Audio Processing
- **Server-side**: FFmpeg, libsndfile
- **Client-side**: Web Audio API, Tone.js
- **Codecs**: libmp3lame (MP3), libflac (FLAC), libvorbis (OGG)
- **Streaming**: HLS/DASH (live)

## DEVELOPMENT RULES

### Before implementing EVERY major feature:

1. **Explain the architecture**
   - How does it fit into the system?
   - What existing components does it use?
   - What new components does it create?

2. **Explain the tradeoffs**
   - Why this approach vs. alternatives?
   - Performance implications?
   - Maintenance implications?
   - Scalability implications?

3. **Recommend the preferred solution**
   - Clear rationale for recommendation
   - Identified risks and mitigations
   - Success criteria

4. **Then implement**
   - Write maintainable, modular code
   - Generate production-ready implementations
   - Update documentation synchronously with code

### Code Quality Standards

- TypeScript strict mode mandatory
- All public APIs documented
- No console logs in production code
- Error handling at system boundaries
- Database migrations tracked in version control
- Infrastructure as Code (Docker, CI/CD)

### Documentation Standards

- Documentation lives alongside code
- Every feature has a README section
- Architecture decisions recorded in DECISIONS.md
- API documented with OpenAPI/Swagger
- Database schema documented
- Deployment processes documented
- Monitoring and alerting documented
